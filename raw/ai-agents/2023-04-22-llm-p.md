## 摘要

LLM 有出色的零样本泛化，但**不能可靠地求解长程机器人规划问题**；经典规划器一旦给定格式化问题即可用高效搜索快速找到正确甚至最优计划。本文提出 **LLM+P**——首个把经典规划器之长融入 LLM 的框架：输入规划问题的自然语言描述，返回用自然语言表达的正确（或最优）计划。做法是先把语言描述转成 **PDDL**（规划领域定义语言）文件，再用经典规划器快速求解，最后把解译回自然语言。论文还定义了一组取自机器人规划场景的多样基准。实验表明 LLM+P 能为多数问题给出最优解，而 LLM 单独连可行解都给不出多数问题；并让家用机器人据自然语言完成复杂操作任务。

## 分章节总结

### I 引言

- LLM 擅长 **linguistic competence（怎么说）**，但弱于 **functional competence（说什么）**；易被训练语料外的算术或新规划问题难住。
- 给出 GPT-4 在 5 积木 Blocksworld（P1）上失败的例子（生成的步骤不合法）。
- 既然已有计算器、通用符号规划器能保证正确答案，自然思路是把 LLM 连接到这些工具。目标：首次让 LLM **正确**求解规划问题，且**不改动 LLM 本身（不微调）**。LLM+P 让 LLM：(1) 输出适合通用规划器输入的问题描述；(2) 用通用规划器求解；(3) 把规划器输出转回自然语言（或接机器人动作执行器）。
- 局限：本文不让 LLM 自行识别"该用 LLM+P 流水线处理"——这是有价值的未来方向。

### II 背景

- **经典规划问题**：$\langle\mathcal{S},s^{init},\mathcal{S}^G,\mathcal{A},f\rangle$（状态空间、初始状态、目标状态集、符号动作集、状态转移函数）。解为符号计划 $\pi=\langle a_1,\dots,a_N\rangle$，逐步满足前置条件、终态满足全部目标条件。
- **PDDL**：经典规划问题的标准化编码，分两文件——**domain 文件**（提升表示：谓词定义状态空间、动作及其前置/效果即转移函数）与 **problem 文件**（对象、初始状态 $s^{init}$、目标条件 $\mathcal{S}^G$）。本工作假设 domain 规则（domain PDDL）已由用户/专家给出。

### III 方法

- **LLM 作 PDDL Writer**：LLM 不擅长规划但擅长描述与翻译，把规划提示改写为 PDDL 本质是"机器翻译"任务。无 prompt-engineering 时 GPT-4 生成的 problem 文件语法对但会用杜撰谓词（如 `empty`）、漏初始条件（如 b1 在桌上）。
- **In-Context Learning**：给 GPT-4 一对"自然语言描述↔PDDL"作上下文后，生成的 problem PDDL 可被规划器直接求解。
- **LLM+P 流程**：假设每个领域有人类专家提供固定的 domain PDDL（动作前置/效果，任务无关）+ 一个简单问题的（描述, problem PDDL）最小示例。对新问题 $P$，LLM 用 in-context learning 推断其 problem PDDL → 连同 domain PDDL 喂给任意经典规划器生成 PDDL plan → LLM 把 plan 译回自然语言。
- **三条假设**：机器人能据对话判断何时触发 LLM+P；提供 domain PDDL 定义可执行动作（任务无关）；提供一个简单问题的自然语言描述及其 problem PDDL。

### IV 相关工作

经典规划算法概览；用 LLM 做规划的近期进展；用外部模块增强 LLM 的研究。

### V 实验

回答四问：LLM-as-P 行不行（完全不行）；LLM+P 比 LLM-as-P 好多少（好很多）；上下文的作用（关键）；能否让服务机器人更高效（能）。

- **基准**：取自历届国际规划竞赛的 7 个机器人规划领域（Blocksworld、Barman、Floortile、Grippers、Storage、Termes、Tyreworld），每领域自动生成 20 个任务。每问题配自然语言描述 + 真值 problem PDDL；每领域配示例描述、PDDL、计划描述作上下文；domain PDDL 由用户/专家预先给出。
- **设置**：全程 gpt-4（temperature 0，确定性）；PDDL 喂 fast-downward 规划器，试 `seq-opt-fdss-1`（保证最优）与 `lama`（不保证最优），搜索上限 200 秒；报最优 alias 成功率，超时则括号内给次优 alias 成功率。还评估 Tree-of-Thoughts 改 BFS 版作 LLM-as-P (ToT)。
- **结果（表I）**：LLM-as-P（含 ToT）几乎全面失败——给的自然语言计划多不可行（不会推理前置条件），复杂空间关系领域（Floortile/Termes/Storage）完全失败；ToT 因 LLM 调用过多多数超时、不适合长程问题。LLM+P 为多数问题产最优解，失败多因 problem 文件误设（漏初始条件致问题不可解）；**无上下文时 LLM 无法产出正确 problem PDDL，上下文对 LLM+P 至关重要**。
- **机器人演示**：真实机器人整理家务（把芥末瓶从咖啡桌移到食品柜、把空汤罐丢进回收箱）。因侧桌与回收箱在路上，最优计划是先带瓶到侧桌、丢汤罐后再取瓶，总代价 22；LLM+P 找到此最优计划，LLM-as-P 给次优计划（先送瓶到食品柜再返回取汤罐，代价 31）。

### VI 结论与未来

用经典规划器赋能 LLM 的最优规划能力，关键设计是让 LLM 专注"把规划问题从自然语言翻译为结构化 PDDL"，并需给 LLM 一个简单（问题, PDDL）对作 in-context 演示。未来：让 LLM 自动检测何时如何用 LLM+P；减少对人类信息的依赖（可能含微调）。
